# Сценарий защиты Mighty Miners (5–7 минут)

## 1. Проблема — 30 секунд

«Tea Local уже поставил товар, но сеть оплатит его через 60–90 дней. Mighty Miners не выдаёт деньги сам: он собирает подтверждённый пакет и передаёт его лицензированному финансовому партнёру.»

## 2. Главная — 45 секунд

Откройте `/`.

- Покажите четыре KPI и адаптивный список заявок.
- Объясните спокойные статусы: жёлтый — ждём сеть, зелёный — проверка, синий — передано партнёру.
- Уменьшите окно до мобильного: таблица становится карточками, снизу появляется навигация и центральная кнопка `+`.

## 3. Новая заявка — 90 секунд

Откройте `/applications/new`.

1. Выберите сеть, введите сумму и накладную.
2. Покажите автоматический расчёт 90 дней.
3. Загрузите три обязательных файла; продемонстрируйте замену или удаление.
4. Проверьте summary и отправьте.

Скажите: «Черновик сохраняется после каждого изменения. В demo хранится имя файла; production‑адаптер отправит бинарный файл в приватный Supabase Storage.»

## 4. Подтверждение сети — 60 секунд

Откройте заявку №125 и нажмите «Открыть как сеть».

- Route `/confirm/supply-125` не показывает кабинет поставщика.
- Регистрация сети не нужна.
- После «Подтвердить поставку» статус общей заявки изменится.

Для production поясните: demo‑token читаемый; настоящий token будет случайным, ограниченным по времени и одноразовым, в базе хранится hash.

## 5. Расчёт и передача — 60 секунд

Вернитесь в `/applications/125`.

- Покажите предварительно доступные 1 940 822 ₸.
- Раскройте «Как рассчитана сумма?».
- Нажмите «Передать заявку партнёру».
- Покажите, что без checkbox кнопка disabled.
- Подтвердите передачу и покажите новый timeline.

Ключевая формулировка: «Окончательные условия определяет партнёр; у нас нет кнопки “Получить деньги”.»

## 6. Техническая защита — 90 секунд

- Next.js App Router разделяет routes поставщика и внешнее подтверждение.
- TypeScript union `ApplicationStatus` не допускает неизвестный статус.
- UI работает через `ApplicationRepository`: localStorage сегодня, Supabase завтра.
- Данные вынесены в `lib/demo-data.ts`, логика — из презентационных компонентов.
- Проверки: `npm run typecheck`, `npm run lint`, `npm run build`, Playwright smoke test.

## Ответы на вероятные вопросы

**Почему localStorage?** Для быстрого полностью кликабельного MVP без внешнего секрета. Граница repository уже выделена.

**Где безопасность документов?** В demo бинарные файлы не хранятся. В production — приватный Storage bucket, RLS, signed URLs и audit log.

**Кто принимает решение?** Только банк или факторинговая компания. Mighty Miners подготавливает и передаёт пакет.

**Это скоринг?** Нет. Расчёт демонстрационный и прямо маркирован предварительным.

**Как исключить повторное подтверждение?** Production‑token имеет expiry, hash в базе и `used_at`; server action атомарно проверяет и закрывает token.
