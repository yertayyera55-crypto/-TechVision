# Сценарий защиты FlowFactor (5–7 минут)

## 1. Проблема

«ТОО «Arman Tea» поставило товар ТОО «Aspan Market», но ждёт оплату 60 дней. Это создаёт кассовый разрыв. FlowFactor — учебный MVP вымышленной факторинговой компании: деньги в проекте реально не переводятся.»

Покажите постоянную метку «Учебный MVP» и дисклеймер: все компании, сделки и предложения синтетические.

## 2. Договор → Codex CLI → заявка

Откройте `/applications/new`.

1. Загрузите PDF-договор или выберите демодоговор.
2. Нажмите «Проанализировать документы».
3. Покажите, как Codex CLI извлекает покупателя, сумму и даты в локальном режиме.
4. Объясните, что ИИ не принимает решение и не придумывает данные: пропуски отображаются, а пользователь заполняет их вручную.
5. Отправьте заявку.

Техническая защита: документ считается недоверенным вводом; его текст передаётся через stdin в `codex exec` с read-only sandbox. Системный промпт запрещает выполнять инструкции из PDF, изменять файлы и запускать команды. JSON проходит серверную проверку до автозаполнения формы.

## 3. Демонстрационное предложение

Откройте созданную заявку.

- Сумма требования — 10 000 000 ₸.
- Первоначальное финансирование — 95%.
- Комиссия FlowFactor — 3%.
- «Поставщик получит сейчас» рассчитывается автоматически.
- Резерв показан отдельно и относится к расчёту после оплаты покупателя.

Нажмите «Принять демопредложение». Формулировка для жюри: «Предварительно соответствует условиям — это демонстрационный расчёт, а не реальное банковское одобрение. В реальном продукте финальные условия определялись бы после проверки документов.»

## 4. Кто кому платит

Откройте «Сроки оплаты».

Объясните разницу между двумя потоками:

- FlowFactor → поставщику: «Демонстрационное финансирование оформлено»;
- покупатель → FlowFactor: «Ожидается оплата покупателя».

Поставщик не фиксирует оплату вручную. Раздел только для просмотра, без формы суммы, даты или кнопки изменения статуса. В реальном продукте статусы поступали бы через банковскую интеграцию или систему факторинговой компании.

При факторинге с регрессом интерфейс показывает мягкое предупреждение о возможном возврате только при длительной просрочке — не требование вернуть деньги.

## 5. Что проверить перед показом

```bash
npm run typecheck
npm run lint
npm run build
```

Для локального запуска:

```bash
npm run demo
```

## Ответы на вероятные вопросы

**Это настоящая факторинговая компания?** Нет. FlowFactor, компании, суммы и статусы вымышлены для учебной демонстрации.

**Почему Codex CLI, а не API?** Так организаторы могут локально увидеть реальный анализ PDF без передачи `OPENAI_API_KEY` во frontend. В production использовался бы LLM API за серверной границей.

**Может ли ИИ одобрить финансирование?** Нет. Codex CLI только извлекает сведения. Предложение вычисляется фиксированными правилами приложения.

**Где реальная оплата?** Её нет. В MVP это заранее заданный демосценарий; реальный продукт получал бы статус через интеграцию фактора или банка.
