# FlowFactor — учебный MVP факторинга

**Учебный MVP.** FlowFactor — вымышленная факторинговая компания, созданная для демонстрации проекта на хакатоне Tech Vision. Все компании, сделки, документы и финансовые предложения в проекте синтетические. Сервис не оказывает реальные финансовые услуги, не переводит деньги и не гарантирует финансирование.

## Какую проблему решает проект

Малый поставщик часто ждёт оплату крупного покупателя 60–90 дней после поставки и испытывает кассовый разрыв. В MVP FlowFactor поставщик загружает договор, система извлекает сведения, показывает понятное демонстрационное предложение и даёт проследить срок, в который покупатель должен оплатить требование фактору.

## Быстрый запуск

Требуется Node.js 20.16+ (или Node.js 22.3+ для `pdf-parse`).

```bash
npm run demo
```

На macOS можно открыть `Mighty Miners.command` двойным кликом. Повторный запуск не создаёт второй Next.js-сервер: скрипт использует уже работающий экземпляр. Чтобы остановить сервер, нажмите `Ctrl+C` в терминале, который его запустил.

Ручной запуск:

```bash
npm install
npm run dev
```

Откройте адрес, который покажет Next.js (обычно `http://localhost:3000`).

## Главный демосценарий

1. На стартовом экране создайте локальный учебный кабинет или нажмите «Продолжить с демо-профилем».
2. Откройте `/applications/new` и загрузите PDF-договор или выберите демодоговор.
3. Нажмите «Проанализировать документы»: локальный Codex CLI извлечёт покупателя, номер договора, сумму, условия оплаты и предмет поставки.
4. Откройте автоматически заполненную «Анкету FlowFactor». Компания берётся из профиля, договорные поля — из PDF, расчёт предложения — из правил приложения. У каждого поля показан источник.
5. Заполните только поля с пометкой «Нужно заполнить», при желании скачайте анкету `.doc` и нажмите «Подтвердить и отправить в FlowFactor».
6. FlowFactor показывает внутреннюю демонстрационную проверку и **предварительное демонстрационное предложение**. Покупателю ничего не отправляется.
7. Примите демопредложение и откройте «Сроки оплаты», чтобы увидеть, когда покупатель должен перечислить оплату **FlowFactor**, а не поставщику.

### Синтетическая сделка

- поставщик: ТОО «Arman Tea»;
- покупатель: ТОО «Aspan Market»;
- денежное требование: 10 000 000 ₸;
- срок оплаты: 60 дней;
- первоначальное финансирование: 95% (9 500 000 ₸);
- комиссия FlowFactor: 3% (300 000 ₸);
- демосумма поставщику сейчас: 9 200 000 ₸;
- резерв после оплаты покупателя: 500 000 ₸;
- тип: факторинг с регрессом.

Это расчёт учебного сценария, а не кредитное или банковское одобрение. В реальном продукте финальные условия определялись бы после проверки документов и договора.

## Страницы

| Route | Назначение |
|---|---|
| `/` | Главная и синтетическая активная сделка |
| `/applications/new` | Договор → AI-анализ → анкета FlowFactor → внутренняя проверка → предложение |
| `/applications/[id]` | Заявка, демопредложение или активная сделка |
| `/payments-monitoring` | «Сроки оплаты» — read-only наблюдение за оплатой покупателя фактору |
| `/deals/[id]/monitoring` | Детали срока оплаты и возможного регресса без редактирования |

## Codex CLI: локальный анализ договора

Для анализа не используется прямой OpenAI API и не нужен `OPENAI_API_KEY`. Серверный endpoint `/api/analyze-contract` принимает PDF до 10 МБ, проверяет MIME-тип, расширение и сигнатуру, извлекает текст и запускает установленный локально Codex CLI в read-only sandbox.

```bash
codex --version
npm run dev
```

Текст договора передаётся через stdin, не попадает в shell-команду и не записывается в обычные логи. Системный промпт прямо запрещает выполнять инструкции из документа, изменять файлы и запускать команды; ответ проверяется по JSON-схеме до автозаполнения формы. Codex CLI только извлекает данные — решение о демопредложении и расчёт выполняются детерминированной логикой приложения.

Если CLI недоступен, форма остаётся рабочей: пользователь может заполнить данные вручную, а интерфейс не показывает технические ошибки CLI. Сканированные PDF без текстового слоя не поддерживаются: MVP честно сообщает об этом без OCR.

## Учебная регистрация и анкета FlowFactor

В начале работы отображается локальная учебная регистрация: название компании, контакт и email хранятся только в `localStorage` этого браузера. Для защиты доступна кнопка «Продолжить с демо-профилем» — она сразу открывает синтетический кабинет ТОО «Arman Tea». Пароли, настоящая аутентификация и серверное хранение аккаунтов не реализованы.

После анализа PDF открывается «Анкета FlowFactor». Она получает данные компании из профиля, сведения о поставке из договора и расчётные суммы из правил MVP. Поля без достоверного источника остаются пустыми с меткой «Нужно заполнить». Анкету можно скачать в `.doc`, но это необязательное действие: заявка отправляется прямо из интерфейса.

В Vercel Codex CLI обычно недоступен. Поэтому deployed-версия показывает прозрачный демосценарий и все остальные страницы, а настоящий AI-анализ проверяется локально на компьютере с авторизованным Codex CLI.

## Сроки оплаты и статусы

Раздел «Сроки оплаты» предназначен только для наблюдения. После финансирования покупатель перечисляет оплату FlowFactor; поставщик не вводит сумму или дату оплаты и не меняет статусы вручную. Ручная функция «Зафиксировать оплату», её модальные окна, поля и связанный код удалены. Запрос покупателю на подтверждение не создаётся автоматически: дополнительную проверку при необходимости инициирует FlowFactor внутри своего процесса.

В демосценарии используются статусы:

- FlowFactor → поставщику: «Предложение сформировано», «Ожидается принятие», «Демонстрационное финансирование перечислено»;
- покупатель → FlowFactor: «Ожидается оплата покупателя», «Срок оплаты приближается», «Оплата просрочена», «Оплата получена фактором», «Сделка закрыта».

При факторинге с регрессом интерфейс предупреждает, что при длительной просрочке FlowFactor **может** потребовать возврат финансирования по договору. Это не требование вернуть деньги. В реальном продукте статусы поступали бы через банковскую интеграцию или внутреннюю систему факторинговой компании.

## Ограничения MVP

- нет реальных переводов, банковских интеграций, скоринга и юридически значимой ЭЦП;
- документы хранятся локально в IndexedDB браузера после загрузки, а не в облаке;
- данные и статусы демонстрационные, синхронизации между устройствами нет;
- FlowFactor не является настоящей финансовой организацией.
